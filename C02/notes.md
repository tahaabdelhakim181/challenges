# Version Control — My Notes

## Definition
A version control system (VCS) records changes to files over time so you can
recall specific versions later. It acts like a time machine for your code.

## Why use it?
- Avoid losing work by accident
- Track who changed what and when
- Collaborate with teammates without overwriting each other
- Roll back to a stable version when bugs appear

## Git vs other VCS
Git is distributed — every developer has a full copy of the history locally,
unlike centralized systems (SVN) where history lives on one server.
