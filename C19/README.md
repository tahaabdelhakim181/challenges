# C19 — AI + Git responsable (2026)

Using AI tools responsibly while maintaining clean Git practices.

## My approach
I used AI to help generate boilerplate code, then reviewed every line
before staging and committing. Each commit message honestly describes
what was done and notes AI involvement where relevant.

## Key rules I applied
- Never commit AI output blindly
- Review, test, then commit
- Write clear commit messages: what changed and why
- Keep commits atomic — one idea per commit
